GitHub

يتكامل SlopCode مع سير عمل GitHub لديك. اذكر /slopcode أو /sc في تعليقك، وسيقوم SlopCode بتنفيذ المهام داخل GitHub Actions runner لديك.

الميزات

  • فرز Issues: اطلب من SlopCode الاطلاع على Issue وشرحها لك.
  • إصلاح وتنفيذ: اطلب من SlopCode إصلاح Issue أو تنفيذ ميزة. وسيعمل على فرع جديد ويقدّم PR يضم كل التغييرات.
  • آمن: يعمل SlopCode داخل GitHub runners لديك.

التثبيت

شغّل الأمر التالي داخل مشروع موجود في مستودع GitHub:

سيأخذك هذا خلال تثبيت GitHub app، وإنشاء workflow، وإعداد secrets.

الإعداد اليدوي

أو يمكنك إعداده يدويًا.

  1. تثبيت GitHub app

    انتقل إلى github.com/apps/slopcode-agent. تأكد من تثبيته على المستودع الهدف.

  2. إضافة الـworkflow

    أضف ملف الـworkflow التالي إلى .github/workflows/slopcode.yml في مستودعك. تأكد من ضبط model المناسب ومفاتيح API المطلوبة ضمن env.

  3. تخزين مفاتيح API ضمن secrets

    في settings الخاصة بالمؤسسة أو المشروع، وسّع Secrets and variables في الشريط الجانبي الأيسر ثم اختر Actions، وأضف مفاتيح API المطلوبة.

الإعدادات

  • model: النموذج الذي سيستخدمه SlopCode. يأخذ الصيغة provider/model. هذا مطلوب.

  • agent: الـagent الذي سيتم استخدامه. يجب أن يكون agent أساسيًا. يعود افتراضيًا إلى default_agent من الإعدادات أو "build" إذا لم يُعثر عليه.

  • share: هل تتم مشاركة جلسة SlopCode. القيمة الافتراضية true للمستودعات العامة.

  • prompt: موجه مخصص اختياري لتجاوز السلوك الافتراضي. استخدمه لتخصيص كيفية معالجة SlopCode للطلبات.

  • token: GitHub access token اختياري لتنفيذ عمليات مثل إنشاء التعليقات، وcommit للتغييرات، وفتح Pull Requests. افتراضيًا يستخدم SlopCode installation access token الخاص بـSlopCode GitHub App، لذا تظهر commits والتعليقات وPull Requests وكأنها صادرة من التطبيق.

    بدلًا من ذلك، يمكنك استخدام GITHUB_TOKEN المدمج في GitHub Action runner دون تثبيت SlopCode GitHub App. فقط تأكد من منح الأذونات المطلوبة في الـworkflow:

    يمكنك أيضًا استخدام رموز الوصول الشخصية(PAT) إن فضّلت ذلك.

الأحداث المدعومة

يمكن تشغيل SlopCode عبر أحداث GitHub التالية:

نوع الحدثيتم تشغيله عبرالتفاصيل
issue_commentتعليق على Issue أو PRاذكر /slopcode أو /sc في تعليقك. يقرأ SlopCode السياق ويمكنه إنشاء فروع، وفتح PRs، أو الرد.
pull_request_review_commentتعليق على أسطر كود محددة في PRاذكر /slopcode أو /sc أثناء مراجعة الكود. يستلم SlopCode مسار الملف وأرقام الأسطر وسياق الـdiff.
issuesفتح Issue أو تعديلهاتشغيل SlopCode تلقائيًا عند إنشاء Issues أو تعديلها. يتطلب إدخال prompt.
pull_requestفتح PR أو تحديثهتشغيل SlopCode تلقائيًا عند فتح PRs أو مزامنتها أو إعادة فتحها. مفيد للمراجعات الآلية.
scheduleجدول يعتمد على Cronتشغيل SlopCode وفق جدول. يتطلب إدخال prompt. يذهب الناتج إلى logs وPRs (لا يوجد Issue للتعليق عليها).
workflow_dispatchتشغيل يدوي من واجهة GitHubتشغيل SlopCode عند الطلب عبر Actions tab. يتطلب إدخال prompt. يذهب الناتج إلى logs وPRs.

أمثلة

إليك بعض الأمثلة على كيفية استخدام SlopCode في GitHub.

  • شرح Issue

    أضف هذا التعليق داخل GitHub Issue.

    سيقرأ SlopCode كامل النقاش، بما في ذلك جميع التعليقات، ويرد بشرح واضح.

  • إصلاح Issue

    داخل GitHub Issue، اكتب:

    وسيُنشئ SlopCode فرعًا جديدًا، وينفّذ التغييرات، ويفتح PR يتضمنها.

  • مراجعة PRs وإجراء تغييرات

    اترك التعليق التالي على GitHub PR.

    سينفّذ SlopCode التغيير المطلوب ويعمل له commit إلى نفس PR.

  • مراجعة أسطر كود محددة

    اترك تعليقًا مباشرة على أسطر الكود في تبويب “Files” داخل PR. يكتشف SlopCode تلقائيًا الملف وأرقام الأسطر وسياق الـdiff لتقديم ردود دقيقة.

    عند التعليق على أسطر محددة، يستلم SlopCode:

    • الملف الدقيق الذي تتم مراجعته
    • أسطر الكود المحددة
    • سياق الـdiff المحيط
    • معلومات أرقام الأسطر

    يسمح ذلك بطلبات أكثر تحديدًا دون الحاجة لذكر مسارات الملفات أو أرقام الأسطر يدويًا.
