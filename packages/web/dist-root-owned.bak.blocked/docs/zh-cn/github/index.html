GitHub

SlopCode 可以与你的 GitHub 工作流集成。在评论中提及 /slopcode 或 /sc，SlopCode 就会在你的 GitHub Actions 运行器中执行任务。

功能特性

  • 问题分类：让 SlopCode 调查某个 Issue 并为你做出解释。
  • 修复与实现：让 SlopCode 修复 Issue 或实现某个功能。它会在新分支中工作，并提交包含所有变更的 PR。
  • 安全可靠：SlopCode 在你自己的 GitHub 运行器中运行。

手动设置

你也可以手动进行设置。

  1. 安装 GitHub App

    前往 github.com/apps/slopcode-agent，确保已在目标仓库中安装该应用。

  2. 添加工作流

    将以下工作流文件添加到仓库的 .github/workflows/slopcode.yml 中。请确保在 env 中设置合适的 model 及所需的 API 密钥。

  3. 将 API 密钥存储到 Secrets 中

    在你的组织或项目的 Settings 中，展开左侧的 Secrets and variables，然后选择 Actions，添加所需的 API 密钥。

配置

  • model：SlopCode 使用的模型，格式为 provider/model。此项为必填。

  • agent：要使用的代理，必须是主代理。如果未找到，则回退到配置中的 default_agent，若仍未找到则使用 "build"。

  • share：是否共享 SlopCode 会话。对于公开仓库，默认为 true。

  • prompt：可选的自定义提示词，用于覆盖默认行为。可通过此项自定义 SlopCode 处理请求的方式。

  • token：可选的 GitHub 访问 Token，用于执行创建评论、提交变更和创建 Pull Request 等操作。默认情况下，SlopCode 使用 SlopCode GitHub App 的安装访问 Token，因此提交、评论和 Pull Request 会显示为来自该应用。

    你也可以使用 GitHub Action 运行器内置的 GITHUB_TOKEN，而无需安装 SlopCode GitHub App。只需确保在工作流中授予所需的权限：

    如果你愿意，也可以使用个人访问令牌（PAT）。

支持的事件

SlopCode 可以由以下 GitHub 事件触发：

事件类型触发方式详情
issue_comment在 Issue 或 PR 上发表评论在评论中提及 /slopcode 或 /sc。SlopCode 会读取上下文，并可创建分支、提交 PR 或回复。
pull_request_review_comment在 PR 中对特定代码行发表评论在代码审查时提及 /slopcode 或 /sc。SlopCode 会接收文件路径、行号和 diff 上下文。
issuesIssue 被创建或编辑在 Issue 创建或修改时自动触发 SlopCode。需要提供 prompt 输入。
pull_requestPR 被创建或更新在 PR 被打开、同步或重新打开时自动触发 SlopCode。适用于自动化审查场景。
schedule基于 Cron 的定时任务按计划运行 SlopCode。需要提供 prompt 输入。输出会写入日志和 PR（没有 Issue 可供评论）。
workflow_dispatch从 GitHub UI 手动触发通过 Actions 选项卡按需触发 SlopCode。需要提供 prompt 输入。输出会写入日志和 PR。

    当你对特定代码行发表评论时，SlopCode 会接收到：

    • 正在审查的具体文件
    • 特定的代码行
    • 周围的 diff 上下文
    • 行号信息

    这样你就可以提出更有针对性的请求，而无需手动指定文件路径或行号。
