소개

SlopCode는 오픈 소스 AI coding agent입니다. 터미널 기반 인터페이스, 데스크톱 앱, IDE 확장으로 사용할 수 있습니다.

바로 시작해 보겠습니다.

사전 준비

터미널에서 SlopCode를 사용하려면 다음이 필요합니다.

  1. 다음과 같은 최신 터미널 에뮬레이터

    • WezTerm, 크로스 플랫폼
    • Alacritty, 크로스 플랫폼
    • Ghostty, Linux 및 macOS
    • Kitty, Linux 및 macOS

  2. 사용할 LLM 제공자의 API 키

설치

SlopCode를 설치하는 가장 쉬운 방법은 설치 스크립트를 사용하는 것입니다.

다음 명령으로도 설치할 수 있습니다.

  • Node.js 사용

    • npm
    • Bun
    • pnpm
    • Yarn

  • macOS/Linux에서 Homebrew 사용

    최신 릴리스를 사용하려면 SlopCode tap 사용을 권장합니다. 공식 brew install slopcode formula는 Homebrew 팀이 관리하며 업데이트 주기가 더 깁니다.

  • Arch Linux에 설치

Windows

권장: WSL 사용

Windows에서는 Windows Subsystem for Linux (WSL) 사용을 권장합니다. 성능이 더 좋고 SlopCode 기능과의 완전한 호환성을 제공합니다.

  • Chocolatey 사용

  • Scoop 사용

  • NPM 사용

  • Mise 사용

  • Docker 사용

현재 Windows에서 Bun을 사용한 SlopCode 설치 지원은 준비 중입니다.

Releases에서 바이너리를 직접 받아 설치할 수도 있습니다.

구성

SlopCode는 API 키를 설정하면 원하는 LLM 제공자를 사용할 수 있습니다.

LLM 제공자를 처음 사용한다면 SlopCode Zen 사용을 권장합니다. SlopCode 팀이 테스트하고 검증한 모델 목록입니다.

  1. TUI에서 /connect 명령을 실행하고 slopcode를 선택한 다음 slopcode.dev/auth로 이동합니다.

  2. 로그인하고 결제 정보를 추가한 뒤 API 키를 복사합니다.

  3. API 키를 붙여 넣습니다.

또는 다른 제공자 중 하나를 선택할 수도 있습니다. 더 알아보기.

초기화

이제 제공자 구성을 마쳤으니 작업하려는 프로젝트로 이동합니다.

그리고 SlopCode를 실행합니다.

다음 명령을 실행해 프로젝트에서 SlopCode를 초기화합니다.

이 명령을 실행하면 SlopCode가 프로젝트를 분석하고 프로젝트 루트에 AGENTS.md 파일을 생성합니다.

팁

프로젝트의 AGENTS.md 파일은 Git에 커밋하는 것을 권장합니다.

이렇게 하면 SlopCode가 프로젝트 구조와 사용 중인 코딩 패턴을 더 잘 이해할 수 있습니다.

사용법

이제 SlopCode로 프로젝트 작업을 시작할 준비가 되었습니다. 무엇이든 물어보세요.

AI coding agent를 처음 사용한다면 도움이 될 수 있는 예시를 소개합니다.

질문하기

SlopCode에 코드베이스 설명을 요청할 수 있습니다.

팁

프로젝트 파일은 @ 키로 퍼지 검색할 수 있습니다.

이 방법은 직접 작업하지 않은 코드 영역을 이해할 때 유용합니다.

기능 추가

SlopCode에 프로젝트의 새 기능 추가를 요청할 수 있습니다. 다만 먼저 계획을 만들도록 요청하는 것을 권장합니다.

  1. 계획 만들기

    SlopCode에는 변경 작업 기능을 비활성화하고 기능을 구현할 방법만 제안하는 Plan mode가 있습니다.

    Tab 키를 눌러 전환하세요. 화면 오른쪽 아래에서 모드 표시를 확인할 수 있습니다.

    이제 수행하길 원하는 작업을 설명해 보겠습니다.

    SlopCode가 원하는 작업을 이해할 수 있도록 충분한 세부 정보를 제공해야 합니다. 팀의 주니어 개발자에게 말하듯이 설명하면 도움이 됩니다.

    팁

    SlopCode가 원하는 작업을 이해하도록 충분한 맥락과 예시를 제공하세요.

  2. 계획 다듬기

    계획이 나오면 피드백을 주거나 세부 사항을 더 추가할 수 있습니다.

    팁

    이미지를 터미널에 드래그 앤 드롭해 prompt에 추가하세요.

    SlopCode는 제공한 이미지를 스캔해 prompt에 추가할 수 있습니다. 이미지를 터미널에 드래그 앤 드롭하면 됩니다.

  3. 기능 구현

    계획이 충분히 마음에 들면 Tab 키를 다시 눌러 Build mode로 전환하세요.

    그리고 변경을 적용해 달라고 요청하세요.

바로 변경하기

비교적 간단한 변경은 계획을 먼저 검토하지 않고 바로 구현하도록 요청할 수 있습니다.

SlopCode가 올바른 변경을 하도록 충분한 세부 정보를 제공해야 합니다.

변경 되돌리기

예를 들어 SlopCode에 변경을 요청했다고 해보겠습니다.

그런데 원하는 결과가 아니었다면 /undo 명령으로 변경을 되돌릴 수 있습니다.

SlopCode가 방금 적용한 변경을 되돌리고 원래 메시지를 다시 보여줍니다.

여기에서 prompt를 수정해 다시 요청할 수 있습니다.

팁

/undo는 여러 번 연속으로 실행할 수 있습니다.

또는 /redo 명령으로 변경을 다시 적용할 수 있습니다.

공유

SlopCode와의 대화는 팀과 공유할 수 있습니다.

이 명령을 실행하면 현재 대화 링크를 생성하고 클립보드에 복사합니다.

참고

대화는 기본값으로 공유되지 않습니다.

다음은 SlopCode 대화 예시 https://opncd.ai/s/<share-id>입니다.
