GitHub

O slopcode integra-se ao seu fluxo de trabalho do GitHub. Mencione /slopcode ou /sc em seu comentário, e o slopcode executará tarefas dentro do seu runner do GitHub Actions.

Recursos

  • Triagem de problemas: Peça ao slopcode para analisar um problema e explicá-lo para você.
  • Corrigir e implementar: Peça ao slopcode para corrigir um problema ou implementar um recurso. E ele trabalhará em um novo branch e enviará um PR com todas as alterações.
  • Seguro: O slopcode é executado dentro dos runners do seu GitHub.

  3. Armazene as chaves de API em segredos

    Nas configurações da sua organização ou projeto, expanda Segredos e variáveis à esquerda e selecione Ações. E adicione as chaves de API necessárias.

Configuração

  • model: O modelo a ser usado com o slopcode. Tem o formato de provider/model. Isso é obrigatório.

  • agent: O agente a ser usado. Deve ser um agente primário. Retorna ao default_agent da configuração ou "build" se não encontrado.

  • share: Se deve compartilhar a sessão do slopcode. O padrão é true para repositórios públicos.

  • prompt: Prompt personalizado opcional para substituir o comportamento padrão. Use isso para personalizar como o slopcode processa solicitações.

  • token: Token de acesso do GitHub opcional para realizar operações como criar comentários, confirmar alterações e abrir pull requests. Por padrão, o slopcode usa o token de acesso da instalação do aplicativo GitHub slopcode, então commits, comentários e pull requests aparecem como se fossem da aplicação.

    Alternativamente, você pode usar o GITHUB_TOKEN embutido do runner do GitHub Action sem instalar o aplicativo GitHub slopcode. Apenas certifique-se de conceder as permissões necessárias em seu fluxo de trabalho:

    Você também pode usar um token de acesso pessoal(PAT) se preferir.

Eventos Suportados

O slopcode pode ser acionado pelos seguintes eventos do GitHub:

Tipo de Evento Acionado Por Detalhes
issue_comment Comentário em um problema ou PR Mencione /slopcode ou /sc em seu comentário. O slopcode lê o contexto e pode criar branches, abrir PRs ou responder.
pull_request_review_comment Comentário em linhas de código específicas em um PR Mencione /slopcode ou /sc enquanto revisa o código. O slopcode recebe o caminho do arquivo, números das linhas e contexto do diff.
issues Problema aberto ou editado Aciona automaticamente o slopcode quando problemas são criados ou modificados. Requer entrada de prompt.
pull_request PR aberto ou atualizado Aciona automaticamente o slopcode quando PRs são abertos, sincronizados ou reabertos. Útil para revisões automatizadas.
schedule Cron baseado em agendamento Execute o slopcode em um cronograma. Requer entrada de prompt. A saída vai para logs e PRs (sem problema para comentar).
workflow_dispatch Acionamento manual pela interface do GitHub Acione o slopcode sob demanda através da aba Ações. Requer entrada de prompt. A saída vai para logs e PRs.

    Ao comentar sobre linhas específicas, o slopcode recebe:

    • As linhas específicas de código
    • O contexto do diff ao redor
    • Informações sobre números de linha
